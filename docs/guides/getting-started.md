# Getting started

Ten przewodnik pomaga szybko zrozumieć, czym jest to repozytorium dziś i jak wejść do niego bez zgadywania.

## 1. Najpierw zrozum, czym to repo jest

`deving` jest obecnie:

- szkieletem repozytorium bez kodu produktu,
- rozwijanym fundamentem dokumentacyjnym,
- bogatym szablonem, który ma ułatwić dalszą rozbudowę,
- miejscem, gdzie najpierw porządkujemy strukturę, nazewnictwo i przepływ informacji.

`deving` nie jest jeszcze:

- aplikacją,
- starterem pod konkretny stack,
- gotowym frameworkiem wdrożeniowym.

## 2. Przeczytaj dokumenty w dobrej kolejności

Zalecana kolejność:

1. `/README.md`
2. `/docs/project/source-of-truth.md`
3. `/docs/project/project-status.md`
4. `/docs/guides/choose-your-path.md`

Jeśli przejmujesz pracę po kimś innym, włącz na początku także:

5. `/docs/project/agent-handoff.md`

## 3. Zdecyduj, po co tu jesteś

Najczęstsze cele:

- chcesz wykorzystać repo jako punkt startowy,
- chcesz zbudować bardzo mocny README,
- chcesz uporządkować dokumentację projektu,
- chcesz przygotować repo do współpracy z AI,
- chcesz później rozszerzyć to pod konkretny stack.

Od celu zależy, które dokumenty są dla Ciebie kluczowe.

## 4. Poznaj trzy warstwy repo

### Warstwa wejściowa
- `README.md`

### Warstwa zarządzania stanem
- `docs/project/*`

### Warstwa instrukcyjna i szablonowa
- `docs/guides/*`
- `docs/templates/*`

## 5. Zasada pracy z tym repo

Najpierw korzystasz z wersji bogatej, a dopiero później upraszczasz.

To oznacza:

- nie usuwaj dokumentów od razu,
- najpierw przeczytaj, do czego służą,
- wybieraj świadomie, które elementy zostają,
- sprawdzaj, czy po uproszczeniu nie zniknęły ważne źródła prawdy.

## 6. Jeśli chcesz zacząć coś zmieniać

Zanim zrobisz pierwszą zmianę:

1. sprawdź `project-status.md`,
2. sprawdź `agent-handoff.md`,
3. upewnij się, że rozumiesz bieżący następny krok,
4. wybierz jeden mały fragment pracy,
5. po zmianie zaktualizuj status i handoff.

## 7. Najczęstszy błąd na starcie

Najczęstszy błąd to potraktowanie tego repo jak pustego startera do szybkiego nadpisania wszystkiego. To repo ma działać inaczej:

- ma Ci najpierw dać bardzo dużo kontekstu,
- dopiero potem pozwolić go świadomie przyciąć,
- ma budować jakość dokumentacji od pierwszych kroków.

## 8. Gdzie iść dalej

- jeśli chcesz wybrać najlepszą ścieżkę użycia, przejdź do `/docs/guides/choose-your-path.md`,
- jeśli chcesz szybko dostosować repo pod siebie, przejdź do `/docs/guides/customization-guide.md`,
- jeśli chcesz pracować z AI/agentami, przejdź do `/docs/guides/ai-agent-guide.md`.
